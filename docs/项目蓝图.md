# AuroraVoyage AI 旅行规划师 - 项目蓝图

## 1. 项目定位
AuroraVoyage 是一款面向 Z 世代与亲子人群的 Web 版 AI 旅行规划师，延续 example 实现的语音交互、AI 规划、费用管理与云同步能力，在前端体验上打造“光感旅图”视觉：浅色渐变背景 + 圆角浮层 + 地图内容为中心的双栏布局，与 example 项目偏深色、shadcn/ui 风格的界面形成清晰区分。

## 2. 项目目标
- **功能对等**：保留语音/文本输入、AI 行程生成、费用预算、用户账户、地图导航与数据可视化等核心能力。
- **体验升级**：提供简洁、亮色、元素更少的 UI，突出「行程时间线 + 费用雷达 + 动态地图」三大核心视图。
- **可运维性**：保持 Supabase + LLM + 讯飞 + 高德 的云原生架构，新增更明确的环境变量与服务抽象层，简化部署。
- **交付完整度**：在开发前输出全套文档，确保需求、架构、测试、计划可被复用到后续实现。

## 3. 关键特性概览
| 模块 | 功能 | 说明 |
| --- | --- | --- |
| 语音与输入 | 讯飞实时语音 + 文本输入 | 统一写入对话上下文，支持多语言唤醒词 |
| AI 行程规划 | DeepSeek/Aliyun 百炼 可互换 | Prompt 工程抽离为配置，支持异常重试与自检 |
| 预算管理 | 语音记账 + 分类预算 + 预警 | 结合 Supabase Edge Functions 计算剩余预算 |
| 用户系统 | Supabase Auth + 行程多版本 | 引入「旅伴共享」角色（扩展性预留） |
| 地图与导航 | 高德 JS + Web 服务 API | 时间线与地图联动，高亮当前日程 |
| 数据可视化 | Recharts + 自适应布局 | 重点显示费用雷达、预算进度、足迹热力 |

## 4. 差异化设计要点
- **色彩与布局**：浅色渐变（#F8FBFF → #D6EEFF）背景，内容以卡片浮层呈现；导航与主要操作按钮使用玻璃拟态投影，区别于 example 的深色侧边栏布局。
- **地图优先**：首页左侧 60% 区域固定显示地图，右侧 tab 在「行程 / 费用 / 记账 / 语音」之间切换，强调“边看地图边编辑”。
- **语音动效**：语音输入按钮采用环形脉冲动效，配合简化的声波动画。
- **组件策略**：减少 shadcn/ui 依赖，采用 Tailwind + 自建组件库，必要时引入 Radix UI 基础交互，确保风格统一。

## 5. 成功指标 (KPIs)
- 行程生成成功率 ≥ 98%（含自动重试）
- 语音识别准确率 ≥ 95%（以中文普通话为基准）
- 新建行程首屏加载 ≤ 3s（移动端网络）
- 关键路径自动化测试覆盖 ≥ 80%
- Docker 镜像冷启动 ≤ 40s

## 6. 依赖与约束
- 仍需获取 DeepSeek/百炼、讯飞、高德、Supabase 等 Key；在 UI 中提供 Key 填写入口。
- 需兼容 Chrome / Edge / Safari / 移动端 WebView。
- 交付物需含 README、Docker 镜像、运行脚本与 PDF（含 repo & 文档链接）。
