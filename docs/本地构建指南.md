# AuroraVoyage 本地构建指南

1. **准备环境**
   - Node.js 18.18+ / 20 LTS（建议 nvm 控制版本）
   - npm 9+ 或 pnpm 8+（脚本默认使用 npm）
   - Git / Docker（可选）

2. **克隆与初始化**
```bash
git clone <your-repo-url>
cd AI-Travel-Planner
```

3. **脚本化配置**
```bash
./scripts/setup.sh
```
- 自动检查 Node 版本
- 缺失 `.env` 时复制 `.env.example`
- 执行 `npm install`

4. **开发模式**
```bash
./scripts/dev.sh
```
默认监听 `http://localhost:3000`，如需自定义端口可修改 `.env` 中的 `NEXT_PUBLIC_APP_URL`。

5. **运行校验**
```bash
./scripts/check.sh
```
顺序执行 `npm run lint` 与 `npm run typecheck`。

6. **Docker 构建（可选）**
```bash
docker-compose up --build
```
使用 `docker-compose` 自动挂载源代码，便于热点调试。

> 若在 WSL1 环境遇到 Node 路径问题，请升级至 WSL2 或在宿主机直接运行脚本。
